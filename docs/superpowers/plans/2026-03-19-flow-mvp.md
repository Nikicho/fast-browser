# Flow MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local JSON-backed `flow` system with `save`, `list`, and `run` commands for sequential command orchestration.

**Architecture:** Store flow definitions under `adapters/<site>/flows/*.flow.json`, validate them in a dedicated flow service, and execute them through the existing `CommandRouter`. Limit MVP step types to adapter `site` commands plus builtin `open`, `wait`, and `waitForSelector` to keep the execution model simple and stable.

**Tech Stack:** TypeScript, Commander, Vitest, existing browser runtime and command router
